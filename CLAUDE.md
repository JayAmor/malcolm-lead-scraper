# Jay's Sandbox — ReadTomato Dev Environment

## Who You Are Helping
Jay (jay@readtomato.com) — Executive assistant to Jim, lead cultivation specialist, and one of the original architects of the HTE (Hot Take Engine) from a prompt engineering and folder management perspective. Deep knowledge of how the content pipeline was designed.

## How To Behave
- Jay has foundational knowledge of the HTE system and prompt architecture. Respect that expertise.
- Explain what you're building and why, so he can connect it to what he already knows.
- When he describes a process he manages manually, help him automate it.
- Always confirm before deleting files or making destructive changes.
- Commit to git frequently with clear messages.

## What Jay Might Build
- Lead cultivation tracking dashboards
- Prospect follow-up automation tools
- HTE prompt testing and comparison tools
- Executive briefing generators (summarize daily activity for Jim)
- Task and calendar management tools
- Client onboarding progress trackers
- Folder/file organization utilities

## Rules
1. This is a sandbox — never connect to production ReadTomato systems
2. Use test/dummy data, never real client data
3. When something works, explain WHY it works
4. If Jay wants to build a web app, use Python (Flask) or simple HTML/JS
5. Keep it simple — working beats perfect

## Tech Stack Available
- Python 3 with pip and venv
- Node.js 20
- Git for version control
- Any pip/npm packages you need — just install them
